
PDV Desktop (Electron) — Arquitetura e Especificação

Objetivo
Desenvolver um sistema PDV desktop usando Electron, focado em:
- Cadastro completo de produtos
- Gestão de estoque
- Sistema de vendas (PDV)
- Sistema de caixa (abertura, sangria, fechamento)
- Cadastro de clientes e fornecedores
- Funcionamento 100% offline
- Sincronização opcional com Supabase
Neste momento não haverá módulo fiscal.

Stack Tecnológica
Desktop: Electron
Frontend: React + Vite + Typescript
UI: Tailwind + shadcn/ui (opcional)
Banco local: SQLite
Biblioteca recomendada: better-sqlite3
Sincronização online: Supabase (Postgres)

Arquitetura do Sistema
Modelo Offline First
O sistema deve funcionar completamente offline.

Fluxo de dados:
PDV (Electron)
↓
SQLite local (fonte da verdade)
↓
Sync Engine
↓
Supabase (replicação online)

Regras principais
1. Toda operação grava primeiro no SQLite
2. Nunca depender da internet para registrar vendas
3. Supabase funciona apenas como backup e sincronização
4. IDs devem ser UUID para evitar conflitos

Regras de Negócio

Finalizar venda (transação):
1 criar venda
2 criar venda_itens
3 registrar pagamentos
4 registrar movimento de estoque
5 registrar auditoria
Se falhar → rollback total
